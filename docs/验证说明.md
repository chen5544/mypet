## 素材处理说明

原视频通过本地 `ffmpeg-static` 检测和抽帧。视频编码为 H.264 `yuv420p`，没有真实 alpha 通道，所以透明背景不是读取原片透明度得到的，而是通过绿幕抠图生成。

抠图策略在 `scripts/process-video.mjs` 中：

- 以绿幕基色 `[56, 166, 35]` 为中心，结合颜色距离和绿色优势判断背景。
- 对接触阴影增加一条偏绿阴影识别通道。
- 对可见绿色像素做第二道硬清理，直接透明化。
- 对半透明边缘做 green spill suppression，减少绿边。
- 对主体内部的小型透明连通区域做邻近像素填补，修复个别帧身上的空缺像素点。
- 当前版本偏向“尽可能去掉绿色残留”，会牺牲一点边缘平滑度。

重新生成素材：

```bash
npm run process:video
```

如果换视频，直接传入视频路径：

```bash
node scripts/process-video.mjs "/path/to/new-video.mp4"
```

奔跑、吃饭、玩耍、跳跃、睡觉使用同一套动作素材处理脚本。动作视频同样是 H.264 `yuv420p`，没有真实 alpha，因此透明帧由绿幕抠图生成，并额外硬清理所有明显绿色像素、修补主体内部小透明洞。

当前向左行走素材来自微信竖屏视频，编码为 HEVC `yuv420p`、没有真实 alpha；生成时先使用 `crop=720:720:0:240` 裁出中间方形区域，避开上下空绿屏和水印，再进行绿幕抠图。

重新生成动作素材：


## 方向锚点

锚点定义在 `src/angle-config.js`：

- `HEAD_CENTER = { x: 314, y: 202 }`
- `IDLE_RADIUS = 120`
- `IDLE_FRAME = 117`
- `ANGLE_KEYS` 和 `ANGLE_ANCHORS` 使用屏幕角度：0 度是鼠标在猫头正上方，顺时针增加。

当前锚点不是按 169 帧均匀摊到 360 度，而是参考 `contact_sheet.png` 和 `head_calibration_sheet.png` 校准：

| 屏幕角度 | 帧号 | 说明 |
| --- | ---: | --- |
| 0 | 169 | 上 / 12 点 |
| 18 | 1 | 闭环进入首帧 |
| 38 | 25 | 右上前侧 |
| 58 | 45 | 右上 |
| 75 | 57 | 右上到右 |
| 92 | 81 | 右 |
| 128 | 97 | 右下 |
| 160 | 109 | 下右 |
| 180 | 117 | 正下 / idle |
| 218 | 129 | 下左 |
| 252 | 141 | 左 |
| 286 | 149 | 左上 |
| 320 | 161 | 左上 / 11 点 |
| 340 | 165 | 左上到上过渡 |
| 360 | 169 | 回到上 |

运行时通过 `frameForAngle(angle)` 在相邻锚点之间插值，得到浮点帧号。渲染层再取相邻两帧做交叉淡入淡出，因此不是 8 方向跳帧。

## 桌宠启动

启动桌宠：

```bash
npm start
```

行为：

- 窗口透明、无边框、置顶。
- 默认出现在主屏幕右下角。
- 默认可拖动。
- 托盘菜单有“锁定并鼠标穿透”和“解锁拖动”。
- 锁定时调用 `setIgnoreMouseEvents(true, { forward: true })`，鼠标事件穿透到其他软件。
- 解锁后可拖动窗口，也可以从托盘菜单移回右下角。
- 鼠标悬浮在宠物身上时显示简洁悬浮菜单，默认包含“奔跑”“吃饭”“玩耍”“跳跃”“睡觉”“行走”。
- 点击动作按钮后循环播放对应 sprite：`cat-run.png`、`cat-eat.png`、`cat-play.png`、`cat-jump.png`、`cat-sleep.png`、`cat-wash.png`。
- 进入任一动作状态后，悬浮菜单会额外显示“跟随”，点击后恢复到初始的跟随光标转头状态。
- 点击动作按钮后，悬浮菜单会隐藏；鼠标离开宠物再重新悬浮时才再次出现。
- 吃饭状态将猫图层放大到 125%，睡觉状态将猫图层缩放到 75%；窗口尺寸和菜单位置保持不变。
- 跳跃状态按 2 倍播放速度循环，让动作节奏更快。
- 行走状态会移动 Electron 透明窗口：从当前屏幕位置向一侧行走，触达显示器工作区边缘后自动反向，使用 `cat-walkRight.png` / `cat-walkLeft.png` 切换方向并循环。
- 状态切换时对猫图层做短促的透明度、阴影和缩放缓动，减少硬切感。
- 猫图层按单帧 `opacity: 1` 显示，不再做双层交叉淡入，以避免主体在动作切换或帧间过渡时显得半透明。

## 回退方式

如果方向不合适，优先修改 `src/angle-config.js` 中的 `ANGLE_ANCHORS`，然后运行：

```bash
npm run validate
npm start
```

如果抠图不合适，调整 `scripts/process-video.mjs` 中的绿幕阈值或阴影阈值，然后运行：

```bash
npm run process:video
npm run validate
```

如果动作抠图不合适，调整 `scripts/process-run-video.mjs` 中的绿幕阈值、阴影阈值或小透明洞修补阈值，然后运行对应动作生成命令：

```bash
npm run process:run
npm run process:eat
npm run process:play
npm run process:jump
npm run process:sleep
npm run process:walk-right
npm run process:walk-left
npm run process:wash
npm run validate
```

如果 Electron 窗口卡住或锁定后不方便操作，从 macOS 菜单栏托盘图标选择“退出”，或者在终端按 `Ctrl+C` 结束 `npm start`。
